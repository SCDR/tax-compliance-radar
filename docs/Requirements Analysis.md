## 2. 项目整体说明

### 2.1 项目背景

东南亚跨境电商企业法务部门面临泰国2026年VAT新政合规痛点，存在法规专业壁垒、语言障碍、审核无标准化流程、合规成本高等问题，本产品通过AI技术提供法规智能解读与合规风险审核能力，赋能企业法务部门提升合规效率。

### 2.2 核心目标

1. 1:1还原初赛方案的两个核心功能演示效果，实现「中文法规智能问答」与「合规风险智能审核」全流程闭环
2. 基于 LlamaIndex + Chroma 实现 RAG 检索增强生成，解决 AI 幻觉问题，保证法规解读的准确性
3. 交付可本地一键启动、无额外部署依赖的演示系统，符合初赛提交要求

### 2.3 MVP 边界（必做，100%覆盖）

1. **核心功能1**：泰国VAT法规智能问答系统，实现中文提问 → RAG精准检索 → 结构化标准回答 → 法规溯源
2. **核心功能2**：泰国VAT合规风险智能审核系统，实现标准化表单录入 → 风险自动识别 → 结构化审核报告生成 → PDF导出
3. **基础辅助功能**：问答历史记录、审核历史记录、强制免责声明全量覆盖

### 2.4 非 MVP 范围（不做，延期）

1. 印尼及其他东南亚国家市场、其他税种的所有内容
2. 用户注册、登录、权限管理、多角色系统
3. 多轮对话、上下文记忆、模糊语义纠错
4. 批量导入、Excel模板、表单暂存、自定义报告模板
5. 政策追踪预警、税额智能计算、法规版本管理
6. 分布式部署、服务器运维、高并发适配
7. 知识图谱、复杂规则引擎、自定义检索策略

## 3. 整体技术架构设计

### 3.1 技术栈明细

| 架构层级 | 技术选型 | 核心职责 |
| :--- | :--- | :--- |
| 前端展示层 | React + Vite + Ant Design + jsPDF | 问答交互页面、审核表单页面、报告展示页面、PDF导出、页面渲染 |
| API 接口层 | Python FastAPI | 接口封装、请求校验、参数处理、跨域处理、结果返回 |
| 业务逻辑层 | Python 原生代码 | 表单校验、硬编码合规规则、历史记录管理、业务流程控制 |
| AI 能力层 | LlamaIndex | 文档加载与分块、向量化处理、RAG检索引擎、Prompt管理、大模型调用、结构化输出控制 |
| 数据存储层 | Chroma 向量数据库 + SQLite 关系型数据库 | Chroma：存储泰国VAT法规文档向量嵌入，提供向量检索能力<br>SQLite：存储业务数据、问答/审核历史、系统配置，免部署免运维 |
| 依赖服务 | 国内大模型API（智谱GLM-4/DeepSeek） + 开源嵌入模型（BGE-zh） | 大模型：结构化内容生成<br>嵌入模型：文本向量化，保证中文检索精度 |

### 3.2 核心数据流

#### 3.2.1 法规智能问答数据流

1. 用户在前端输入中文提问，提交至 FastAPI 接口
2. 后端校验请求合法性，非空校验通过后，调用 LlamaIndex 检索引擎
3. LlamaIndex 将提问文本向量化，调用 Chroma 向量库执行混合检索（向量+关键词），召回相关法规文档片段
4. LlamaIndex 将召回的法规片段 + 固定结构化 Prompt 输入大模型，生成符合要求的结构化回答
5. 后端将回答内容 + 强制免责声明写入 SQLite 问答历史表，通过接口返回给前端
6. 前端渲染展示结构化回答、法规溯源信息、免责声明

#### 3.2.2 合规风险审核数据流

1. 用户在前端填写标准化审核表单，提交至 FastAPI 接口
2. 后端校验表单必填项，格式化业务数据，通过硬编码合规规则引擎完成初步风险识别与分级
3. 后端将业务数据、初步风险结果 + 固定报告 Prompt 输入 LlamaIndex，结合 Chroma 法规知识库补充法规依据与整改建议
4. LlamaIndex 调用大模型生成符合初赛方案模板的结构化审核报告
5. 后端将业务数据、审核报告、风险统计写入 SQLite 审核记录表，通过接口返回给前端
6. 前端渲染展示审核报告，支持 PDF 导出，全页面强制展示免责声明

## 4. 核心业务需求详细设计

### 模块一：泰国VAT法规智能问答系统

**解决痛点**：法务缺乏东南亚税务法规专业知识

#### 4.1.1 底层支撑：泰国VAT法规知识库构建

| 功能项 | 详细需求 | 技术实现映射 | 业务规则 |
| :--- | :--- | :--- | :--- |
| **法规素材整理** | 完成泰国VAT核心法规素材定稿，仅覆盖初赛方案核心场景 | 人工整理为 Markdown 格式，按章节拆分，标注元数据 | 1. 必须覆盖：2026年新政、VAT注册规则、税率体系、进口关税规则、免税政策、违规处罚、平台代收代缴要求<br>2. 每条法规标注：法规名称、发布机构、生效时间、核心条文、官方原文链接<br>3. 演示素材总量控制在10个文档以内 |
| **文档加载与分块** | 自动加载法规文档，完成文本分块与预处理 | LlamaIndex SimpleDirectoryReader + RecursiveCharacterTextSplitter | 1. 分块参数固定：chunk_size=512，chunk_overlap=50，适配法律条文场景<br>2. 自动过滤无效字符、空白内容，保留完整条文语义 |
| **向量库构建** | 完成文档分块的向量化，构建 Chroma 向量索引 | LlamaIndex VectorStoreIndex + ChromaVectorStore | 1. 嵌入模型固定为 BGE-Large-zh，保证中文法律文本检索精度<br>2. 每个向量块绑定元数据：文档ID、章节名称、法规名称、生效时间、原文链接<br>3. 向量库集合名称固定为 thailand_vat_regulations |
| **检索引擎配置** | 配置 RAG 检索策略，保证召回准确率 | LlamaIndex 混合检索引擎 | 1. 检索模式：向量检索 + BM25 关键词检索混合模式<br>2. 召回条数固定为 top4，避免上下文溢出<br>3. 相似度阈值固定为0.7，低于阈值直接返回兜底提示 |

#### 4.1.2 中文问答与结构化输出功能

| 功能项 | 详细需求 | 输入输出标准 | 技术实现映射 | 验收标准 |
| :--- | :--- | :--- | :--- | :--- |
| **提问入口与校验** | 前端提供中文提问输入框，完成基础请求校验 | 输入：用户中文自然语言提问文本<br>输出：请求合法性校验结果 | 前端：Ant Design Input/Search 组件<br>后端：FastAPI 参数校验，非空校验、长度限制（≤500字） | 1. 空提问拦截，提示「请输入您要咨询的泰国VAT合规问题」<br>2. 超长度内容拦截，提示「提问内容不能超过500字」 |
| **结构化回答生成** | 强制按固定结构生成回答，禁止自由输出 | 输入：用户提问文本、Chroma召回的法规片段<br>输出：固定结构包含7个字段 | LlamaIndex 固定 Prompt 模板，强制约束输出结构 | 1. 提问示例输出结构与初赛方案100%对齐<br>2. 所有核心规则均有对应的法规依据，无编造内容<br>3. 关键量化指标正确高亮 |
| **法规溯源能力** | 回答内容自动关联对应的法规原文片段 | 输出：回答中每个核心规则标注对应的法规片段来源 | LlamaIndex 自动绑定召回片段元数据，前端渲染 | 1. 所有核心规则均可追溯到对应的法规原文片段<br>2. 原文片段与回答内容完全匹配，无错位 |
| **问答历史记录** | 自动留存用户的问答记录，支持查看历史 | 输入：问答生成的提问与回答内容<br>输出：按时间倒序排列的问答历史列表 | SQLite 问答历史表存储，FastAPI 提供查询接口 | 1. 自动保存所有成功响应的问答记录<br>2. 历史记录展示提问内容、生成时间，支持查看完整回答 |

**结构化回答字段**：
1. 法规依据：明确标注法规名称、条目编号
2. 核心规则：提炼关键量化指标，高亮展示
3. 合规建议：可执行的操作建议
4. 风险提示：违规后果与处罚规则
5. 操作指引：基础操作步骤说明
6. 法规原文链接：可跳转的官方链接
7. 强制免责声明

## 7. 非功能需求

### 7.1 性能需求

1. 页面加载时长 ≤ 2秒，接口响应超时阈值 ≤ 5秒
2. 单条问答请求响应时长 ≤ 3秒，单条审核请求响应时长 ≤ 4秒
3. 本地运行无卡顿，PDF导出时长 ≤ 2秒
4. 系统本地启动时长 ≤ 10秒，一键启动无额外配置

### 7.2 安全与合规需求

1. **强制合规要求**：所有页面、AI输出内容、审核报告、PDF文件必须全量展示固定免责声明，一票否决
2. **数据安全**：所有数据本地存储，无数据外传，用户业务数据仅本地可见
3. **AI幻觉抑制**：所有AI输出必须基于Chroma知识库内容，无匹配内容时明确提示，禁止编造法规信息
4. **接口安全**：所有接口做参数校验，防止非法注入，无越权访问风险

### 7.3 兼容性需求

1. 前端适配PC端主流浏览器（Chrome、Edge、Firefox），分辨率1920*1080下无布局错乱
2. 系统支持Windows/Mac/Linux本地运行，无操作系统依赖
3. Python版本兼容3.10及以上，无特殊系统依赖包
